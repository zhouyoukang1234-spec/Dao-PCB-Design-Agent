# PCB 全链路实现方案
> **核心理念**：五感以至于无感 — 每步有明确动作，不依赖记忆，不产生决策疲劳

---

## 全链路总览（十环闭合）

```
① 需求锁定 → ② oshwhub复刻 → ③ 原理图修改 → ④ PCB布局布线
      ↑                                                      ↓
⑩ 迭代升级 ← ⑨ 调试验证 ← ⑧ 固件烧录 ← ⑦ 焊接组装 ← ⑤ DRC检查 → ⑥ 打样下单
```

---

## 环节一：需求锁定（5分钟）

**本质**：把想法变成一句话功能描述 + 一张接口表

### 用户当前两条主线

| 项目 | 一句话描述 | 核心接口 |
|------|-----------|---------|
| **P1** STM32点阵板 | STM32F103C6通过串口控制LED点阵显示字符 | USART1(PA9/PA10), 点阵数据线×8, 电源3.3V/5V, SWD调试 |

## 环节三：原理图修改（30~60分钟）

**本质**：在复刻板基础上增删接口，不动电源/晶振/复位等核心电路

### P1 需要修改的地方

| 需增加 | 原因 | 嘉立创EDA操作 |
|--------|------|--------------|
| LED点阵驱动接口（2×8排针） | 连接点阵显示模块 | 放置2.54mm排针封装，连接PA0~PA7 |
| USART1 TX/RX 测试点 | 调试用 | 在PA9/PA10加测试点焊盘 |
| 保留SWD接口(4pin) | 程序烧录/调试 | 原板已有，确认SWDIO/SWCLK/VCC/GND引出 |

| 可删除 | 原因 |
|--------|------|
| 原板多余外设（按照实际需求） | 减小板面积，降低成本 |

### AI辅助：eda-copilot（可选，降低门槛）

```
安装方式：
1. 打开 https://github.com/klxf/eda-copilot/releases
2. 下载最新 .zip 插件包
3. 嘉立创EDA → 设置 → 扩展管理 → 本地安装 → 选zip文件
4. 启用"外部交互权限"
5. 原理图页面 → 顶部菜单 Copilot → 配置API(支持Deepseek/GPT/Claude)

使用示例（对话框输入）：
"帮我在原理图中添加一个LED点阵驱动接口，使用PA0到PA7，2x8排针，请生成网表"
```

**检查点（ERC规则检查）**：
- 菜单 → 设计 → 电气规则检查(ERC)
- 目标：0 Error，Warning可接受
- 常见错误：VCC/GND未连接、引脚悬空 → 加PWR_FLAG或接GND

## 环节四：PCB 布局布线（1~2小时）

**本质**：先定位置（布局），再连线（布线），遵守三条黄金规则

### 三条黄金规则

```
规则1  晶振靠近MCU，走线越短越好，晶振周围包地(GND铺铜)
规则2  去耦电容(100nF)必须紧靠每个VCC引脚，不能绕远
规则3  电源线宽 ≥ 0.5mm，信号线宽 0.2~0.3mm，差分对等长
```

### 嘉立创EDA操作流程

```
Step 1  原理图 → 菜单「设计」→「更新/导入PCB」→ 所有元件飞线出现
Step 2  手动布局：
        - 先放MCU/模组在中间
        - 晶振/复位/去耦电容贴近MCU
        - 接口(USB/排针/调试口)放在板边
Step 3  自动布线(可选起点)：
        菜单「布线」→「自动布线」→ 选Topological模式 → Run
        ⚠️ 自动布线结果必须人工检查，晶振/高速信号需手动重布
Step 4  铺铜：
        快捷键 P → 选铜皮 → 框选全板 → 属性设为GND → 铺铜
        双面铺铜：顶层GND + 底层GND，通过过孔连通
```

**检查点（DRC规则检查）**：
- 菜单 → 设计 → 设计规则检查(DRC)
- 目标：0 Error
- 常见错误：间距不足(<0.1mm) → 移开元件；未连接网络 → 补飞线

## 环节五：DRC 检查 + 导出前确认（15分钟）

**本质**：最后一道关卡，出Gerber前必须清零

### 检查清单（逐项打勾）

```
[ ] ERC 0错误（原理图层）
[ ] DRC 0错误（PCB层）
[ ] 板框(Board Outline)已定义，尺寸合理（P1建议50×40mm，P2建议60×40mm）
[ ] 所有元件丝印清晰，不被焊盘遮挡
[ ] SWD/USB接口方向正确（插件能插进去）
[ ] 3D预览检查：元件无穿插、无悬空
```

## 环节六：Gerber 导出 + 立创打样（15分钟）

**本质**：一键导出标准格式，填好参数下单

### Gerber导出

```
嘉立创EDA → 菜单「制造」→「PCB制板文件(Gerber)」→「导出Gerber」
自动打包为 Gerber.zip，保存到：
  D:\keil代码\[项目名]\Gerber_v1_日期.zip（台式机）
  或 Y:\嘉立创EDA\[项目名]\（笔记本）
```

### 立创商城下单（5片≈20~30元，3~5天到货）

```
1. 打开 https://www.jlcpcb.com （立创捷配/嘉立创PCB）
2. 点击「立即下单」→ 上传 Gerber.zip
3. 参数设置（新手默认即可）：
   - 层数：2层
   - 板材：FR4
   - 厚度：1.6mm
   - 颜色：绿色（最便宜）
   - 表面处理：HASL（喷锡，最便宜）
   - 数量：5片
4. 加入购物车 → 支付（≈20元）→ 等待3~5天
```

**同时操作（等板子期间）**：准备BOM物料，淘宝/立创商城备货

## 环节七：物料准备 + 焊接（板到后2~4小时）

**本质**：对照BOM表逐一核对，按"先矮后高"顺序焊接

### BOM导出

```
嘉立创EDA → 菜单「制造」→「物料清单(BOM)」→ 导出Excel
```

### P1 核心元件清单

| 元件 | 规格 | 数量 | 来源 |
|------|------|------|------|
| STM32F103C6T6 | LQFP-48 | 1 | 立创商城/淘宝 ≈5元 |
| 晶振 8MHz | 贴片 3225 | 1 | 配套电容2×22pF |
| 去耦电容 | 100nF 0402 | ×10 | 每个VCC引脚一个 |
| 复位电路 | 100nF+10K | 1套 | 原板已有 |
| SWD调试接口 | 2.54mm 4pin | 1 | 用于烧录 |
| LED点阵接口 | 2.54mm 2×8排针 | 1 | 接现有点阵模块 |

### 焊接顺序（五感变无感）

```
① 贴片电阻/电容（最矮，先焊）
② QFP封装MCU（难度最高，热风枪+助焊剂）
③ 模组(ESP32-WROOM等，排针型直接插焊)
④ 排针/排座（最后，最高）
⑤ 焊后清洗助焊剂（酒精棉）
```

## 环节八：固件烧录（30分钟）

**本质**：代码→编译→通过SWD接口写入Flash

### P1：STM32 Keil烧录流程

```
工具：Keil MDK5 + ST-Link或DAP-Link仿真器

1. 打开 D:\keil代码\stm32\stm32点阵.uvprojx
2. 菜单 Project → Rebuild All（确认0 Error）
3. 仿真器接线：
   ST-Link/DAP-Link → 板子SWD接口
   SWDIO → PA13, SWCLK → PA14, VCC → 3.3V, GND → GND
4. 菜单 Flash → Download（快捷键 F8）
5. 观察串口输出：波特率115200，发送'H'→点阵显示H
```

### P2：ESP32 Arduino烧录流程

```
工具：Arduino IDE + USB转串口(CP2102/CH340)

1. 打开 D:\电路代码\sketch_sep3b\sketch_sep3b.ino
2. Arduino IDE → 工具 → 开发板 → ESP32 Dev Module
3. 修改代码中WiFi名称/密码：
   const char* ssid = "你的WiFi名";
   const char* password = "你的WiFi密码";
4. 按住BOOT键 → 点Upload → 看到"Connecting..."后松开BOOT
5. 串口监视器(115200) → 看到IP地址 → 浏览器访问验证
```

## 环节九：调试验证（1~2小时）

**本质**：系统化排查，不靠运气

### 三层检查法

```
层1 上电检查（不烧固件，先验电路）
    万用表量VCC→GND：不短路(>1kΩ)
    上电后量3.3V稳压输出：3.28~3.32V ✓
    发烫元件 → 立刻断电，检查短路

层2 固件运行检查
    串口输出正常 → 通信链路OK
    LED闪烁测试 → GPIO驱动OK
    P1: 发'H'到串口 → 点阵显示H
    P2: 访问WiFi IP → 页面出现 → 拉动舵机

层3 功能验证
    P1: 完整点阵字符集测试
    P2: 0°/90°/180° 舵机精度验证，HTTP响应时间<200ms
```

### Proteus仿真（可选，调试难题时用）

```
位置：D:\proteus（台式机，1.6GB完整安装）
用途：在没有实物板的情况下验证STM32逻辑
操作：新建工程 → 添加STM32F103C6 → 加载HEX文件 → 仿真运行
```

## 常见惑祸速解

| 问题 | 根因 | 解决 |
|------|------|------|
| 晶振不起振 | 电容值不匹配(22pF vs 实际需求) | 看芯片手册负载电容，常见8pF/12pF/20pF |
| STM32无法烧录 | BOOT0引脚状态错误 | BOOT0接GND正常运行，接3.3V进串口下载模式 |
| ESP32不连WiFi | SSID/密码错误 或 2.4GHz/5GHz混淆 | ESP32仅支持2.4GHz |
| PCB打样报错 | Gerber文件缺层或板框未闭合 | 重新导出，勾选"检查Gerber"选项 |
| 焊后短路 | QFP引脚连锡 | 吸锡带+助焊剂，热风枪重新回流 |
| DRC报间距错误 | 元件贴太近(<0.1mm间距) | 移开元件，通常SMD间距≥0.1mm，THT间距≥0.2mm |
| ERC报引脚悬空 | 未使用引脚未处理 | 不用的输入引脚接GND或VCC，不用的输出引脚接测试点 |
