## 二、高价值新发现（推荐集成）

### ⭐ kicad-tools (rjwalters) — **最高优先**

> `pip install kicad-tools` | PyPI | MIT | Python 3.10+ | 无需运行KiCad实例

**完整的AI Agent→KiCad工具链**, 覆盖我们pipeline的几乎所有环节:

| 能力 | 说明 | 对应我们的模块 |
|------|------|---------------|
| **Pure Python解析** | 无需KiCad安装即可解析.kicad_sch/.kicad_pcb | `kicad_arm.py` (补充) |
| **A*自动布线** | 内置autorouter, 支持GPU加速(CUDA/Metal) | `kicad_arm.auto_route` (替代BFS) |
| **LLM驱动布局** | `PCBReasoningAgent` — LLM做布局决策, 工具执行几何 | 🆕 新能力 |
| **布局优化** | 力导向+遗传算法, 支持混合优化 | 🆕 新能力 |
| **走线优化** | 路径缩短+过孔减少 | 🆕 新能力 |
| **Pure Python DRC** | 无需kicad-cli, 支持JLCPCB/PCBWay/OSHPark规则 | `pcb_eye.py` (增强) |
| **Circuit Blocks** | MCU/LDO/USB/Crystal等可复用电路模块 | `circuit_dna.py` (增强) |
| **参数化封装生成** | SOIC/QFP/QFN/SOT/DIP/0402等 | 🆕 新能力 |
| **符号库管理** | 编程式创建/编辑.kicad_sym | 🆕 新能力 |
| **Datasheet工具** | 搜索/下载/PDF解析/引脚表提取 | 🆕 新能力 |
| **MCP Server** | `kct mcp serve` — Claude/Windsurf直连 | `pcb_mcp.py` (参考) |
| **制造商规则** | JLCPCB/PCBWay/OSHPark/Seeed DRC profiles | `pcb_jlcpcb.py` (增强) |
| **JSON输出** | 全CLI支持`--format json`, 机器可读 | Agent友好 |

```bash
# 安装
pip install kicad-tools          # 基础 (CPU)
pip install kicad-tools[cuda]    # NVIDIA GPU加速
pip install kicad-tools[all]     # 全部

# CLI示例
kct symbols project.kicad_sch --format json
kct drc board.kicad_pcb --mfr jlcpcb
kct route board.kicad_pcb
kct reason board.kicad_pcb --auto-route
kct placement board.kicad_pcb --optimize
kct datasheet search STM32F103C8T6
kct mcp serve
```

**集成方案**: `pip install kicad-tools` → 在 `pcb_pipeline.py` 中作为可选后端调用 → 替代BFS布线, 增加LLM布局/布局优化/走线优化/Pure Python DRC

### ⭐ KiCAD-MCP-Server (mixelpixx) — v2.2.3

> 122工具 | 16分类 | TypeScript+Python | JLCPCB 2.5M+零件库 | MIT

最成熟的KiCad MCP Server实现:

| 亮点 | 说明 |
|------|------|
| 122个MCP工具 | 覆盖项目/板/元件/布线/DRC/导出/原理图/库/自动布线 |
| JLCPCB API集成 | 2.5M+零件库, 实时价格/库存, Basic/Extended分类 |
| 动态符号加载 | ~10,000个KiCad符号, 零配置 |
| 智能布线系统 | 自动引脚发现+旋转感知, 正交/直连/智能路由 |
| FreeRouting集成 | Java/Docker/Podman三种方式 |
| IPC API后端 | KiCad 9实时UI同步 (实验性) |
| Tool Router模式 | 智能分类发现, AI上下文减少70% |

**集成价值**: 参考其JLCPCB API实现 (JLCSearch API by tscircuit) 增强我们的 `pcb_jlcpcb.py`; 参考其Tool Router模式优化 `pcb_mcp.py`

### kicad-mcp-python (Finerestaurant) — IPC-API路线

> KiCad官方IPC-API | kicad-python | 最稳定的AI↔KiCad通道

| 亮点 | 说明 |
|------|------|
| 官方IPC-API | 通过 `kicad-python` (PyPI) 与运行中的KiCad通信 |
| ActionFlowManager | 每次操作前后自动截图对比验证 |
| PCB对象操作 | 创建/修改/移动/旋转/删除 footprint/track等 |
| Board Status | `get_board_status` 全局状态+截图 |

**前置条件**: 需要编译安装 `kicad-python` (Git submodule), KiCad需开启IPC Server

**集成方案**: 等 `kicad-python` 稳定发布PyPI后, 作为 `kicad_native.py` 的替代方案。目前我们的L2 CLI降级策略仍然有效。

## 四、行业趋势 (2025-2026)

- **AI布局**: 考虑布线结果的智能布局 (kicad-tools已实现LLM-driven)
- **AI布线**: 主流EDA工具集成AI布线 (Altium 2025已有AI routing suggestions)
- **LLM设计助手**: 理解上下文的设计助手 (KiCAD-MCP-Server 122工具)
- **代码化PCB**: SKiDL + atopile + kicad-tools 三条路线并行
- **云端EDA**: flux.ai等云平台崛起, 但开源+本地仍是我们的主线
