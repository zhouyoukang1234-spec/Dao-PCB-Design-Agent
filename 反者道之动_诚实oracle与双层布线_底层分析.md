## 0. 一句话结论

上一阶段系统自报 **21/21 自由能=0、全部真实交付**。
用反演造出的**诚实 DRC（R007 走线短路 / R008 net 全连通）**一照，

## 1. 反演方法：为什么"自由能=0"不可信

预测编码框架里，`free_energy = Σ(precision_i × error_i)`，
系统在 `free_energy=0` 时宣称"实质闭环"。但这有一个**致命循环**：

> **自由能只在它的 oracle（DRC 规则集）能看见的维度上为零。**
> oracle 看不见的缺陷，对自由能贡献为 0——系统因此"自信地错着"。

旧 DRC 只有 6 条规则（R001 焊盘重叠 / R002 元件出界 / R003 重号 /
R004 net 无 segment / R005 焊盘坐标重合短路 / R006 钻孔间距）。
它**看不见**两类最致命的布线缺陷：

| 盲区 | 后果 | 真实工具（KiCad/Altium）必查 |
|---|---|---|
| 异网**走线-走线**铜箔重叠 | 短路 | ✅ clearance/short |
| 异网**走线压在焊盘**铜箔上 | 短路 | ✅ |
| net 上多焊盘是否被铜**真正全连通** | 开路 | ✅ connectivity/ratsnest |

旧布线器 `auto_route_simple()` 的注释写着"双层路由 F.Cu/B.Cu+过孔"，
**但实现是纯单层 F.Cu**，且带 Level-3/4 降级：
- Level-3：日志打 `⚠️ 可能产生clearance违规` 但照样布；
- Level-4：日志打 `❗ 可能短路` ——**明示自己在制造短路**，只为凑"0 未连接"。

单层栅格上给多网布线，在拓扑上对稍密的板**本就不可能平面化**（无法避免交叉），
于是降级穿铜成了常态。旧 oracle 看不见，自由能照样归零。**这就是反演要揭穿的那一面。**

## 2. 本阶段做了什么（从根因，非臆造）

### 2.1 造出诚实 oracle（`kicad_origin/engine/drc.py`，+R007/R008）
- **R007 走线 clearance/短路**：同层异网两走线，净间距 = 中心线最近距离 −(w_a+w_b)/2；
  `< -短路容差` → 短路(ERROR)，`< 最小间距` → clearance(WARNING)。
  另判走线中心线进入异网焊盘铜箔范围 → 短路(ERROR)。**按铜层判定**（F/B 分层，
  通孔 `*.Cu` 占双层），避免跨层几何假阳性。
- **R008 net 全连通**：并查集（DSU）把 pad 中心、走线端点、过孔点并通；
  同 net 的 pad 落在 >1 连通分量 → 开路(ERROR)。
- 经 `run()` 的 `_r*` 自动发现机制注册，无需改 runner。

### 2.2 真双层迷宫布线（`pcb_brain/kicad_arm.py::auto_route_simple` 重写）
- **状态 = (gx, gy, layer)**，F.Cu=0 / B.Cu=1；
  平面移动代价 1，**换层（过孔）代价 8**（抑制过孔泛滥）——统一代价搜索（Dijkstra）。
- **焊盘解析提取铜层**：SMD 只封锁自身层，通孔 `*.Cu` 封锁双层，
  SMD 焊盘只能在自身层起步/落地。
- **不同网走线落在不同层即不短路**——这正是真实 PCB 用 ≥2 层消除交叉的根本机制。
- **诚实降级**：找不到双层路径时**留开路（R008/R004 显形）**，
  不再 Level-4 强行穿铜。宁可诚实开路，不制造短路。

### 2.3 诚实基线（同一 R007/R008 oracle 下）

| 指标 | 旧单层布线器 | 新双层布线器 | 变化 |
|---|---|---|---|
| 全量隐性 error | **8191** | **144** | **-98.2%** |
| 真正零错模板 | 0 | **3** | ams1117 / nrf52840 / stm32h743 |

## 4. 残余根因（反演的下一层）：栅格分辨率 vs 焊盘间距

剩余 R007 多为"走线压在异网焊盘铜箔上"，集中在 0.5mm 间距的 LQFP/QFN 旁。根因：

1. **栅格太粗**：GRID=0.25mm，0.5mm 间距焊盘相邻仅差 2 格，
   一个走线格的物理宽度（0.25mm 线 + 间距）就可能盖住相邻脚。
2. **端点豁免邻格**：为让走线能离开焊盘，src/dst 周围 3×3 格被豁免封锁，
   细间距下这会顺带豁免**邻脚（异网）**，走线得以压上去。
3. **栅格迷宫的本质局限**：均匀栅格无法表达"焊盘逃逸（escape routing）"的
   亚栅格几何——这是 BGA/QFN 扇出（fanout）必须解决的问题，
   工业工具用**无栅格（gridless / shape-based）或 push-and-shove** 布线器。

这一层不是调参能根治的，需要架构升级（见 §6 路线图 P1）。

## 5. 对照工业与前沿（定位差距）

### 5.1 vs KiCad / Altium / Cadence
- **连通性模型**：工业工具有完整 ratsnest + 连通图，DRC 与布线器共享同一几何/电气模型；
  本项目 DRC（kicad_origin）与布线器（kicad_arm）是**两套独立几何**，
  这正是"自由能虚高"能发生的结构性土壤——**oracle 与执行器没有共享真值**。
- **布线器**：工业用 gridless / push-and-shove（KiCad 的 router、Altium ActiveRoute）；
  本项目是均匀栅格 Lee 迷宫——对细间距逃逸天然吃力。
- **kicad_origin 休眠**：仓库里有 ~17K 行纯 Python KiCad 引擎（零外部依赖），
  却长期只被当 DRC 用，未参与布线/连通真值。**这是最大的未动用杠杆**。

### 5.2 vs AI+EDA 前沿
- Google **AlphaChip / 强化学习布局**、Nvidia **GPU 加速 P&R**、
  学界 **DeepPCB / 生成式布线**：核心范式是
  **"可微/可学习的代价场 + 大规模搜索"**，而非硬编码降级规则。
- 本项目的预测编码框架（free_energy 作为统一误差信号）**天然契合**这一范式：
  把 R007/R008 的 error 当作可反传的"惊异"，让布线策略**朝降低自由能演化**——
  这是把"无为而无不为"落到算法的最自然路径。

### 5.3 vs 预测编码神经科学（Friston / Rao & Ballard）
- **主动推断（active inference）**：误差不只被动度量，还驱动**行动**去消除它。
  当前 R008 开路是"认知误差"（系统知道自己没连上），
  正确的主动推断是**回到布线器换层/绕行重试**，而非止步报告。
- **精度加权（precision weighting）**：不同 net（电源/高速差分/普通信号）
  应有不同 precision；当前所有 net 等权，未利用这一维度。

## 6. 分层路线图（按自由能杠杆排序）

> 原则：哪条根因贡献的自由能最大，先打哪条（无为而无不为：让解从框架物理里自然涌现）。

### P0 — 已完成（本阶段）
- ✅ 诚实 oracle（R007/R008），揭穿 8191 隐性 error
- ✅ 真双层布线 + 过孔，8191 → 144（-98.2%），3 板真正零错

### P1 — 细间距焊盘逃逸（吃掉残余 58% 的集中误差）
- **焊盘感知的端点豁免**：豁免只放行**本网**焊盘格，绝不豁免异网邻脚；
  配合**亚栅格逃逸短线**（pad → 第一个安全栅格点的专用 fanout 段）。
- **自适应栅格**：在 IC 焊盘区域局部加密（GRID→0.1mm），空旷区保持 0.25mm。
- 预期：drone_aerial / drone_flight / lcd_tft 的 R007 大幅下降。

### P2 — 主动推断式重试（吃掉 40 处 R008 开路）
- net 布线失败时，不止步报告，而是**触发重排序 + 强制换层 + 拆已布线重布（rip-up & reroute）**，
  把 R008 开路当作驱动**行动**的误差信号——真正的 active inference。

### P3 — 统一真值（根除"自由能虚高"的结构土壤）
- 让布线器与 DRC **共享 kicad_origin 的同一几何/连通模型**，
  唤醒休眠的 17K 行引擎。oracle 与执行器同源，自由能不再能虚高。

### P4 — 可学习代价场（对接 AI+EDA 前沿）
- 把 free_energy 设为布线代价场的可调目标，
  用搜索/学习让布线策略朝"全局最低自由能"演化（精度加权区分电源/高速/普通网）。

### P5 — 自我修复闭环
- pipeline 末端：DRC error→定位→局部 rip-up→重布→再 DRC，
  直到 `free_energy=0`（**这次是诚实的 0**）或收敛到不可约残差并诚实报告。
