## 1. 自下而上的结构解构

### 1.1 六层架构 (真实 import 关系属实)

```
Layer 0  _pcb_bootstrap.py   UTF-8/路径/环境根基          —— 真实有效
Layer 1  circuit_dna.py      21 个 DNA 模板 (128KB)        —— 数据在, 但部分 Comp 损坏(见 1.3)
Layer 2  kicad_arm.py        PCB 生成 / BFS 布线 / DRC      —— 生成依赖外部 KiCad 封装库
Layer 3  pcb_eye/jlcpcb/ibom 感知 / BOM / 交互 BOM         —— "感知"只看文件名/价格, 不看实质
Layer 4  pcb_pipeline.py     6 阶段编排                    —— success 恒为 True (病灶所在)
Layer 5  pcb_core/brain/mcp/server                         —— 门面齐全
旁支     kicad_origin/       纯 Python 制造引擎             —— 能力很强, 但流水线没接它(沉睡)
```

### 1.2 致命的"降级缝" (facade seams, 实测证据)

以 `ams1117_power` 跑一遍旧流水线，逐阶段反演真值：

| 阶段 | 自报状态 | 实测产物 | 真相 |
|------|---------|---------|------|
| DNA | ok | 4 元件 | 真实 |
| PCB 生成 | ok | `(pad ` 计数 = **0** | KiCad 封装库不在场 → `_parse_fp_pads` 返回空 → 焊盘全无 |
| 布线 | `routed=0` 仍记 ok | 0 段走线 | 无焊盘 → 无网络端点 → 无可布之线 |
| DRC | ok, violations=0 | status=`no_kicad_cli` | 根本**没跑** DRC, 0 是"没看" |
| Gerber | ok | `G04 Mock Gerber F_Cu*` | **占位文件**, 非真实铜层 |
| 汇总 | **success=True** | — | `pcb_pipeline.py:330` 硬编码 `success=True` |

根因链：**封装库缺位 → 0 焊盘 → 0 布线 → DRC 空过 → Gerber 造假 → 总报告说成功**。
任意一环都"没抛异常"，所以系统**自认为成功**。

### 1.3 附带发现的数据缺陷

- 部分 DNA 模板的 `Comp` 定义参数错位：`fp_lib` 落入分组名、`fp_name` 落入坐标元组
  (如 `passive:(78.0, 44.0)`)，这些元件**根本没有合法封装**。
- `stm32f103c6_dot_matrix` 接入真实 DRC 后报 **21 处违规**——以前被"跳过"掩盖了。

## 2. 为什么用预测编码 (Predictive Coding) 重构

人脑不靠"动作执行完了没有"来确认世界，而靠 **预测 → 观测 → 预测误差** 的循环
(Rao & Ballard 1999；Friston 自由能原理 / Active Inference)：

1. **生成模型自上而下预测**：脑预测下一刻的感官输入；
2. **自下而上观测**：真实感官回来；
3. **只有预测误差 (surprise) 才向上传播**，驱动下一步；
4. **两条最小化路径**：
   - 改世界 (action / pragmatic)：动手让现实符合预测；
   - 改认知 (perception / epistemic)：去采样、补足观测以消除"我不知道"；
5. **精度加权**：越可信/越关键的误差，权重越高；
6. **自由能 = Σ 精度·误差**，趋零即"模型与世界一致"。

把它映射到 PCB Agent：

| 预测编码 | PCB Agent |
|---------|-----------|
| 生成模型 (先验) | DNA 模板 = 设计意图 |
| 自上而下预测 | 由 DNA 推出产物应有的不变量 (焊盘数/网络/必需走线/可制造) |
| 自下而上观测 | **从产物文件反演真值**, 不信任任何"status=ok"自述 |
| 预测误差 | 预测 vs 观测之差 |
| 实质误差 (pragmatic) | 产物确实错 → 去修正产物 (补焊盘/补布线) |
| 认知误差 (epistemic) | 工具缺位看不见 → 去补足观测手段 (跑真实引擎) |
| 自由能=0 | 实质闭环, 真正可交付 |

**旧系统恰恰缺第 2~4 步**：从不比较"预测产物"与"实际产物"，所以永远无 surprise、
无反馈、无法自我纠偏，只能一路 mock 到底还自称成功。

## 3. 本次落地 (给系统装上误差信号 + 两次"动作")

### 3.1 `pcb_predict.py` — 预测编码核验引擎 (核心)

- 由 DNA **自上而下预测**：封装数、应接网焊盘数、网络数、必需走线数；
- 从 `.kicad_pcb` / `gerber/` / `pipeline_report.json` **自下而上反演**真值；
- 计算每个"预测单元"的精度加权 **预测误差 (surprise)**，汇成 **自由能**；
- 把误差分类为 **实质 (去修正产物)** / **认知 (去补足观测)**，
  并输出**主导误差对应的下一步动作** (active inference：最小化最大 surprise)；
- `delivered = (自由能 == 0)`——**这才是"真正交付"的定义**，取代旧的 `success=True`。

### 3.2 `footprint_pads.py` — 第一性原理焊盘生成 (动作①·改世界·pragmatic)

> 反者道之动：与其依赖外部 KiCad 封装库在场，不如对**几何确定的标准封装**
> 直接由 land pattern 第一性原理生成焊盘。

- 覆盖：片式无源 (0201~1210 的 R/C/L/LED/Fuse)、SOT-223、排针 (PinHeader NxM)；
- **诚实留白**：复杂 IC (QFP/QFN/SOIC/BGA/模组) 几何依赖器件手册 → **故意返回空**，
  让 `pcb_predict` 继续把它标为认知误差，而**不伪造**一个"看起来完整"的假焊盘
  (绝不重蹈 mock 的覆辙)。

### 3.3 `fab_origin.py` — 激活沉睡的纯 Python 制造引擎 (动作②·补观测·epistemic)

- 仓库本就有 `kicad_origin/engine`：`Board.load → run_drc / write_gerber / write_excellon`，
  **零第三方依赖**，却从未被流水线调用；
- 本桥在 `kicad-cli` 不在场时，用它产出**真实** Gerber/钻孔并跑**真实** DRC，
  把"认知误差(看不见)"变成"真实观测(看得见)"，无需安装 KiCad。

### 3.4 流水线集成 (非破坏)

`pcb_pipeline._finalize` 现额外输出 `delivered / free_energy / delivery_next_action / surprises`，
并诚实打印"阶段执行 ✅"与"实质交付 ❌/✅"两条**互不替代**的结论。
`_stage_drc / _stage_gerber` 在无 KiCad 时优先走 `kicad_origin`，mock 仅作最后兜底。

### 3.5 实测：自由能被一步步"动作"压到 0

以 `ams1117_power` 为例 (全程**未安装 KiCad**)：

```
重构前        自由能 54  (焊盘0/9 · 布线0 · DRC认知盲 · Gerber造假)
+内置焊盘     自由能 27  (焊盘9/9 ✅ · 布线26段 ✅ · 剩 DRC/Gerber 认知误差)
+纯Python引擎 自由能 0   ✅ 真实交付 (真实DRC 0违规 · 真实Gerber 11文件/2铜层)
```

这就是预测编码循环的完整演示：**预测→观测→误差→动作→再观测**，
实质误差用"改世界"消除，认知误差用"补观测"消除，自由能趋零方称交付。

## 4. 全量诚实记分牌 + 误差分类 (21 模板, 重构后)

`python pcb_predict.py --all` 实测：**真实交付 2/21**。其余 19 个不再是笼统的"差"，
而是被预测编码引擎**自动归入 4 个确定的误差类**，每类都有明确的下一步：

| 类 | 模板 (数量) | 误差性质 | 下一步动作 |
|----|------------|---------|-----------|
| **A 已实质交付** | `ams1117_power`, `led_indicator` (2) | 自由能=0 | 推进真实打样核验 |
| **B 缺 bespoke 封装几何** | `ch32v003` `esp32_servo` `lora` `motor_driver` `nrf52840` `rp2040` `smartwatch` `stm32g031` `usb_c_pd` `w5500` `drone_aerial` (11) | 实质·pad gap (大部分焊盘已生成, 缺晶振/JST/USB-C/SOT-23-5/二极管/按键/电感等异形封装) | 扩展 `footprint_pads` 异形几何 或 接入真实 KiCad 封装库 |
| **C 真实 DRC 违规** | `drone_flight_controller`(26) `gd32f103`(21) `stm32f103`(21) (3) | 实质·几何重叠/间距 (焊盘已全, 接入真实 DRC 后暴露) | 修布局间距 / 焊盘重叠 |
| **D DNA 数据损坏** | `esp32s3_rs485_can` `stm32h743_core` `industrial_power` `lcd_tft_43` `safety_protection` (5) | 实质·根因在数据层 (100% Comp 的 `fp_name` 是坐标元组, 且全部重叠于一点) | 修 `circuit_dna.py` 这些模板的 Comp 定义 |

> 关键不在"还是 2/21"，而在**误差从不可见变得精确可寻址**：
> A 类已闭环；B 类只差异形封装几何；C 类的 DRC 违规以前被"跳过"完全看不见，现在有确切计数；
> D 类彻底暴露了一个**数据层根因 bug**(5 个模板的元件全是坏数据，过去却照样报 ✅)。
> 这正是预测编码的价值——系统第一次**知道自己哪里不行、错在哪一层、下一步该做什么**。

### 4.1 本轮已落地的"动作"与新发现

- **扩展 `footprint_pads.py`**：在片式无源/SOT-223/排针之外，新增**双列 (SOIC/SSOP/SO)** 与
  **四边 (QFP/LQFP/QFN/DFN/UFQFPN)** 规则封装的参数化 land pattern——其几何由
  `引脚数 + 间距 + 本体尺寸`(全部编码在封装名里)**完全确定**，是真实计算而非伪造。
  结果：多个含 MCU 的模板从"看不见的 pad 缺口"前进到"看得见的真实 DRC 违规"(C 类)。
- **实践中发现的两个 bug 并已修复**：① `observe_pcb` 误把走线/过孔自带的 `(net ..)`
  计入"接网焊盘"，改为只数 pad 块内的 net 引用；② `builtin_fp_pads` 对损坏的非字符串
  `fp_name` 未设防，已加 `isinstance` 护栏。
- **暴露 D 类数据根因**：5 个模板的 **全部** Comp 都是坏数据，这是过去"23/23 PASS"
  facade 掩盖最深的一处——没有真实误差信号时，连"元件根本没有合法封装"都看不出来。

## 5. 路线图 (按精度加权误差排序的"动作"待办)

1. **扩展 `footprint_pads.py` 的精确几何**到 SOIC/QFP/QFN/SON/晶振/JST/USB-C/按键
   (或：接入真实 KiCad 封装库 / easyeda2kicad 作为封装源)。每补一族，多个模板自由能下降。
2. **修复 `circuit_dna.py` 中错位的 `Comp` 字段** (fp_lib/fp_name 被坐标污染)。
3. **解决 `stm32f103c6_dot_matrix` 的 21 处真实 DRC 违规** (布局间距/焊盘重叠)。
4. **真实打样核验闭环**：自由能=0 后，把 Gerber 送 JLCPCB 实际下单/SMT，回流真板照片/电测，
   作为最高层的预测误差源 (设计预测 vs 实物)。
5. **把预测误差并入 `pcb_self_loop.py`**：让永久自循环以"降低全量自由能"为目标函数，
   优质架构在不断实践中自己生长 (无为而无不为)。
